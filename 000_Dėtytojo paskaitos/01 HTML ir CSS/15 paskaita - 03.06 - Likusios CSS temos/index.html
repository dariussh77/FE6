Git ir Github

Git - kodo versijavimo programa.

GitHub - svetainė, kurioje keliamos kodo versijos.

  1. Įsirašyti Git'ą į kompiuterį
  2. Patikrinti ar git'as įrašytas - terminale parašyti "git --version"
  3. Susikurti gitHub'o paskyrą
  4. Konfiguruoti git'ą
  5. git config --global user.name gitHubVartotojoVardas
  6. git config --global user.email gitHubVartotojoEmail
  7. git config --global core.editor code
  8. git config --global core.autocrlf true | input (true jeigu windows | input jeigu mac/linux)
  9. Patikrinti ar teisingas config failas - git config --global --edit

SASS/SCSS

Suteikia galimybę rašyti CSS'ą naujoviškesniu būdu, kur galima lengvai kurti ir naudoti kintamuosius, nest'inti, rašyti: sąlygas | ciklus | funkcijas | mix'inus ir t.t.
Naudojant kažkurį CSS preprocesorių, reikia parašytą kodą konvertuoti į paprastą CSS, kurį supranta svetainė.

Pros

  • Nereikia/labai mažai rašyti CSS
  • Pagreitina darbą
  • Lengva naudotis
  • Suprogramuoti responsiveness

Cons

  • Daug perteklinio/nepanaudoto kodo
  • Neunikalus dizainas
  • Reikia skirti laiko išmokti naudotis
  • Prastenis google paieškos sistemos įvertinimas

    • CSS Framework - kažkieno kito parašytas CSS kodas, kurį tu gali naudoti.
    Norint naudotis framework'u, reikia jį prijungti prie savo kodo naudojant CDN.
    Dauguma framework'ų veikia suteikiant jų aprašytas klases kažkokiems HTML elementams.
    Yra Classless CSS framework'ų. Juos naudojant, nereikia suteikti HTML elementams klasių. Viskas aprašyta naudojant tag'ų/Atributų/Nth'ų... selektorius.
