Git ir GitHub

Git - kodo versijavimo programa

GitHub - svetainė, kurioje talpinamas versijuotas kodas ir juo dalinamasi tarp programuotojų

Nuoroda į GitHub
  1. Užsiregistruoti į GitHub'ą (susikurti paskyrą)
  2. Prisijungti prie GitHub'o

Komandinės eilutės komandos

  • TAB - užbaigia rašomą eilutę arba pasiūlo variantus (nenaudoti ant tuščios komandinės eilutės)
  • ls - išvardina direktorijas ir failus (list'ina)
  • cd - direktorijos keitimas (change directory)
  • CTRL C - atšaukia vykdomą komandą / stabdo procesus
  • clear - išvalo terminalo eilutes
  • direktorijosKūrimas, failųKūriams, trinimas, perkelimas/pervadinimas, failųKoregavimas
Nuoroda į Git

Git Komandos

  • git init - sukuria git'o repozitoriją
  • git add nurodytiFailai - pridėti koreguotus/naujus failus/aplankalus prie paruoštų failų kodo versijos kūrimui (commit'ui)
  • git commit -m - iš paruoštų (staged) failų/aplankalų sukuriama kodo versija su atitinkama žinute (message (-m)), nusakančia kas toje kodo versijoje pakito
  • git status - grąžina informacija apie dabartinės kodo versijos būklę (statusą)
  • git push - nusiunčia commit'us į išorinę repozitoriją (gitHub'ą)
  • git pull - atsiunčia commit'us iš išorinės repozitorijos (gitHub'o)

Git konfiguracijos

  • git config --global user.name "Tavo GitHub paskyros vardas"
  • git config --global user.email "PrivaluTasPastasKuriuoKureiGitHubPaskyra@gmail.com"
  • git config --global core.editor "code"
  • git config --global core.autocrlf true (vietoje "true" rašyti "input" jeigu linux/mac)
  • git config --global -e
  • git config --help

Klasės (Class)

Klasės yra skirtos generuoti naujus objektus.

Klasė yra objektų kūrimo šablonas.
Klasė privalo būti aprašyta prieš kreipiantis į ją.
Klasės rašomos iš didžiosios reidės.
Klasės konstruktoriuje privalo būti "this".
